## Описание

Harpoon — лёгкая библиотека для языка Goida, предоставляющая удобный HTTP-клиент и привязки к нативной реализации на Rust. Библиотека упрощает выполнение HTTP-запросов из Goida-кода и предоставляет:

- отправку запросов: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`;
- установку заголовков и куки;
- отправку JSON-тела запроса;
- настройку таймаута и прокси;
- объектную обёртку `Ответ` с доступом к `статус`, `тело`, `заголовки`, `куки` и `ошибка`;
- поддержку сессий через `Сессия` для переиспользования экземпляра клиента.

Фактически, Goida-слой предоставляет удобные классы `Клиент`, `Запрос`, `Ответ` и `Сессия`, а нативная часть реализована на Rust (cdylib) и отвечает за выполнение HTTP-вызовов.

## Быстрый пример

Ниже — минимальный пример использования из Goida:

```goida
сес = новый веб.сессия.Сессия()

зап = сес.запрос()
		.ссылка("https://jsonplaceholder.typicode.com/todos/1")
		.заголовки(словарь("Content-Type", "application/json"))
		.ожидание(30)

ответ = зап.получить()
если (!ответ.ошибка()) {
		печать(ответ.статус())
		печать(ответ.данные())
} иначе {
		печать("Ошибка:", ответ.ошибка())
}
ответ.прочитан()
сес.закрыть()
```

Метод `Ответ.данные()` пытается распарсить тело как JSON и вернуть структуру данных, а `Ответ.прочитан()` освобождает системные ресурсы, связанные с ответом.

## Сборка

Сборка нативной части (Rust) создаёт cdylib, который загружается из Goida:

```bash
cargo build --release
# или для отладочной сборки
cargo build
```

Готовая библиотека будет находиться в `target/release/` (или `target/debug/` для отладки). В Goida-примерах используется путь `target/debug/harpoon` для загрузки в режиме отладки.
